# /execute

**Deutsch:** Baue den Plan Aufgabe fuer Aufgabe testgetrieben. Lies
`docs/process/workflow.md` (Execute) und `docs/process/commits.md`. Pro Aufgabe:
Failing Test schreiben, Rot sehen, minimal implementieren, Gruen sehen, dann
einen atomaren Conventional Commit erstellen. Aufgaben so isoliert halten, dass
sie einzeln reviewbar sind.

**English:** Build the plan task by task, test-driven. Read
`docs/process/workflow.md` (Execute) and `docs/process/commits.md`. Per task:
write the failing test, see it fail, implement the minimum, see it pass, then
make one atomic conventional commit. Keep tasks isolated so each is independently
reviewable.

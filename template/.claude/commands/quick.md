# /quick

Use the Tier 1/2 shortcut for small, isolated changes without a
separate plan file. Read `docs/process/workflow.md` (Quick). Before editing:
state goal, touched files, and risk; add a test for Tier 2. Before finishing,
take a light pass over the relevant `docs/process/review-checklist.md` questions
(security sinks, one-owner). Escalate to `/plan` as soon as persistence, auth,
contracts, or external integrations are in scope.

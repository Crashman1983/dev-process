# Architecture Decision Records

An ADR records one significant, hard-to-reverse decision: the context, the choice, and
its consequences. Write one when a decision shapes structure, crosses a boundary, or
future contributors would otherwise ask "why is it done this way?". Use `template.md`.

Every new ADR file must be added to the index below in the same change.
